The App() function in App.js and the definition of a React component

The App() function is a React component, which is a plain JavaScript function that returns some JSX.

A React component has two jobs:

  • Produces a JSX: which tells React what component to display on the screen.
  • Handles UserEvent: For example, when user types some text input, or clicking on a button.


What's a JSX?


JSX is a set of instructions to tell React exactly what content to display on the screen at any given time. JSX syntax is very similar to HTML and follows many similar rules.

React JSX element can be used to create a normal HTML element, or to show another element.

For every JSX, React will loop through the elements within. When React looks at a JSX element, it will ask itself whether that element is a standard DOM element.

If Yes, it will display that element on the HTML.

If No, it will check whether it is a component. If Yes, it will call the component function and inspect all the JSX that it gets back.

Example: Let's look at this line in the return() statement of App():

return (
  <div>
    <Field label="Enter English" onChange={setText} value={text} />
              ...
);

React will ask itself: "Is <div> a standard DOM element?" - Yes. Then it will move on to <Field..> and the answer is No, this is not a standard DOM vanilla element, however it is a component that was created and imported with:

import Field from "./components/field";

What React will do is look at the ./components/field.js file and iterate through the default() function:

import React from "react";

export default ({ label, value, onChange }) => {
  return (
    <div>
      <label>{label}</label>
      <input
        className="input"
        value={value}
        onChange={e => onChange(e.target.value)}
      />
    </div>
  );
};
  • Is <div> a DOM element? Yes -> Display that on the screen.
  • Is <label> a DOM element? Yes -> Display that on the screen.
  • Is <input> a DOM element? Yes -> Display that on the screen.

Then, React will go back to App.js and move on to the <Languages> element. It does this repeatedly until it has walked over all components.

The index.js file


In the public directory, we can see the index.html file that is loaded up on the browser. Then the request is going to go back to our folder and look for additional javascript files. The first file that gets requested and boots up our whole React application is the index.js file which is:

import React from "react";
import ReactDOM from "react-dom";
import App from "./App";

ReactDOM.render(<App />, document.getElementById("root"));

The important line is:

ReactDOM.render(<App />, document.getElementById("root"));

In the render() call. we have a few arguments:

  • <App /> tells React to call the App function and get back the JSX output, then turn it into HTML.
  • document.getElementbyId("root") tells React where to display the App function within the HTML.

index.js: The imported libraries


In index.js we imported two libraries: React and ReactDOM, what are they? It turns out that when we refer to and use React, we actually interact with two seperate libraries.

The React library itself knows how to work with components. It can be referred to as a reconciler.

The ReactDOM knows how to take instructions on what we want to show, and display that by turning it into HTML. It can be referred to as the renderer.
