Short Description

The typical Pandas user learns one dataframe method at a time, slowly scraping features together through trial and error until they can solve the task in front of them. In this tutorial you will re-learn how to think about dataframes from the ground up, and discover how to select intelligently from their abilities to solve your data processing problems through direct and deliberately-chosen steps.

IPython Built-in magic commands

IPython has a set of predefined 'magic functions' that can be calle dwith a CLI style syntax. There are two kinds of magics, line-oriented and cell-oriented

Line Magics are prefixed with %. They work like OS Command line calls. They get as an argument the rest of the line, where arguments are passed without parentheses or quotes. Line magics can return results and can be used in the right hand side of an assignment. In other word, their statement is per line.

Cell Magics are prefixed with double %%. They get as an argument not only the rest of hte line but also the lines below it in a seperate argument. In other word, their statement is per the whole cell.

(https://ipython.readthedocs.io/en/stable/interactive/magics.html#cell-magics)

Series dtype

When working with series, it's important to know the data type of the values first before operating with them.
To find out the data type of the values in a series do Series.dtype.

.head() and .tail()

Using head() method displays the first 5 rows of the dataframe. Similarly you can use tail(). You can specify the number of rows as an argument in the bracket, overriding the default 5.

These are genuine data operations. They do not just display or print the first n rows but really build a new dataframe by snipping off n numbers of rows.

They are data slicing/ reduction operations.

titles.head()  ## This produces a fully functional dataframe

new = titles.head()  ## This new df can be assigned to a variable.

Displaying the dataframe

If we are interested in displaying only one column, put the string of the column name in the sqr bracket.

df['colname']

If we want to see multiple columns, instead of passing just a string, pass a list of string. Note the double square brackets:

df[['col1', 'col2']]

Counting

You can do len(df) to count the number of rows in a df.

To count the number of occurences within a Series, you can use:

Series.value_counts('colname')

Filtering the dataframe

Filtering = reducing the dataframe to only the rows that match our criteria.

We can look at only specific column by: titles['year'] (df = titles, column = year)

We can even manipulate the stdout by doing something like this: titles['year'] + 100 which will return all the years plus 100.

So that was math operations, what's more usefule is bool, which we can do titles['year'] > 1985 which returns rows of True and False.
Important: This returns the full list of rows with the respective boolean value. It does not slice the dataframe yet.

To only show the True values, you can do: titles[titles.year > 1985] or title[titles['year'] > 1985].

You can use & and | for and, or like this:

Show me the movie titles between 1985 and 1987 (notice the brackets)

titles[(titles['year'] >= 1985) & (titles['year'] <= 1987)]

How many movies were made between 1985 and 1987?

len(titles[(titles['year'] >= 1985) & (titles['year'] <= 1987)])

So using dataframe object is similar to using List where as instead of the index number within the square brackets, you use the column name.

Using operational String methods

A lot of the times we want to use common string methods like startswith() or endswith()… The problem is a lot of the string object methods' names coincide with names of the series object.

However there is a little namespace str that is attached with series object that contains all string data operation.

E.g, instead of doing this..

Series.startswith('abc')

.. you do this..

Series.str.startswith('Hamlet')

(More details: https://pandas.pydata.org/pandas-docs/version/0.22/api.html#string-handling)

Pandas Index

Indexing helps with data lookup performance, making it faster.

Both rows and columns can be indexes.

How to index and use indexes

Example: Reduce the dataframe 'cast' to only the row with title = 'Sleuth':

Operation without index:

%%time 
cast[cast.title == 'Sleuth']

CPU times: user 128 ms, sys: 2 µs, total: 128 ms
Wall time: 127 ms

Indexing:

%%time
c = cast.set_index('title')

CPU times: user 496 ms, sys: 64.1 ms, total: 560 ms
Wall time: 357 ms

Operation with index:

%time c.loc['Sleuth']
#+begin_src 
CPU times: user 74.5 ms, sys: 0 ns, total: 74.5 ms
Wall time: 73.7 ms

Multiple column indexing:

c = cast.set_index(['title', 'year'])

Operation with multiple column indexing:

%time
c.loc[('Sleuth',1972)]

CPU times: user 2 µs, sys: 0 ns, total: 2 µs
Wall time: 4.53 µs

When you present to Python a statement with the == operator in a dataframe that has four million rows, you are asking Python 4 millions True or False question.

However with loc[index] it is much easier and take less steps, i.e loc('Sleuth') would skip everything before S.

set_index()

The set_index() method takes a column name argument and make that column the index column.

df.set_index('col')

This is similar to when we pass index_col = 'col' as an argument for read_csv().

To set_index for multiple column, instead of passing a string of the column name, pass a list of column names.

index(append = True)

Passing of the append = True argument in the index() method passes the index as the additional one, on top of the existing one.

reset_index()

This is the direct opposite of set_index(). It takes argument(s) of column name(s) that toss them back to normal data, not index anymore.

c.reset_index('year')

sort_index()

To rearrange the order of the df/ series by index sorting, do:

series.sort_index()

Grouping Pandas dataframe groupby()

df.groupby() takes column name(s) as argument and produces a DataFrameGropBy object that can be used for multiple things.

We can use df.groupby().size() to see the count of different grouping of data. It displayes a hierachical index if multiple column names are pased.

We can use df.groupby().col.max() or min() and even df.groupby().col.mean(), ..etc.. for data science purposes.

See how many roles George Clooney plays per year

c= cast[cast.name == 'George Clooney']
c.groupby('year').size()

See how many roles George Clooney plays in total

c= cast[cast.name == 'George Clooney']
c.groupby('year').size().sum() ## This returns the same result with len(c)

See how many roles George Clooney plays per decade

c.groupby(c['year'] // 10 * 10).size()

To groupby() multiple column, the column names have to be in a list i.e a pair of square brackets.

Unstack operation

By default, unstack() tackles the inner most/ first index, i.e

df.unstack() = df.unstack([1])

unstack pivots a level of the index labels i.e turning a column header into a row header. It allows us to see data next to each other in a row instead of being on the same column.

The opposite of unstack() is stack(). An easy way to remember is that the unstack() method pivots the data UP.

fillna() method

Pandas is reown for not replacing N/A value with zero as in many circumstances, N/A values should be left as Not Available. This is 'NaN' in pandas.

However, if you want to replace NaN values with something, use the fillna() method. Be careful with this as this is dependent on the nature of the data you are dealing with. For example, missing data should not be recorded as '0' in financial report.

Pivot operation

Dataframe has a pivot function.

df.pivot['col1','col2','col3']

It is basically a combination of set_index(), unstack(), and sort_index().
In the above example, col2 will be pivoted upto the top (unstack), col1 over to the left, leaving col3 as the information in the centre.
