Logging Basic

Python comes with a logging module built-in so this is very handy.

Logging is a great way to understand what is happening in our program and in what order it's happening.

Python's logging module makes it easy to create a record of custom messages that we write. The log messages will describe when the program execution has reached the logging function call and list any variables we have specified at that point in time.

On the other hand, a missing log message indicates that a part of our code was skipped and never executed.

Logging is definitely more handy than printing, especially when our projects are larger than basic small projects.

Logging versus print()

Importing logging and setting up the config is somewhat tedious. We may want to use print() instead, however we should not give in to this temptation!

Once we have done with debugging, we'll end up spending a lot of time removing print() calls from our code after each log message. We might accidentally remove a print() call that is legitimately intended for non-log messages.

The nice thing about log messages is that we can fill our program with as many as we like, and we can always disable them all later by adding a single logging.disable(logging.CRITICAL) call. Unlike print(), the logging module makes it easy to switch between showing and hiding log messages.

Log messages are intended for the programmer, not the user. For message that we want the user to see, we should use print(). However, stuff like the contents of some dictionary value or how long it takes to run a function, the users don't really care.

Let's incorporate logging and use it instead of print(). But first, let's talk about logging level.

Logging Level

There are five logging levels:

Level Logging Function Description & When it's used
DEBUG logging.debug() This is the lowest level, it is used for small details. It is typically
    of interest when dianosing problem.
INFO logging.info() Used to record information on general events in our programs or confirm
    that things are working at their point in the program.
    These are confirmations that things are working as expected.
WARNING logging.warning() Used to indicate a potential problem that doesn't prevent the program
    from working but might do so in the future. The software is still working
    as expected.*This is the default level of logging*.
    Example: "Disk Space low"
ERROR logging.error() Used to record an error that caused the program to fail to do something.
    The software has not been able to perform some function.
CRITICAL logging.critical() The highest level. It is used to indicate a fatal errr that has caused or
    is about to cause the programe to stop running entirely.

Our logging message is passed as a string to these functions.

The logging levels are suggestions. Ultimately, it is up to us to decide which category our log message faills into.

The benefit of logging levels is that we can change what priority of logging message we want to see. Passing logging.DEBUG to the basicConfig() function's level keyword argument will show messages from ALL levels (DEBUG is the lowest level). After we develop our program some more, we may be interested only in errors. In that case, we can set the basicConfig() level argument to logging.ERROR. This will show only ERROR and CRITICAL logging message.

What does the root mean?

Because we have not specified a specific logger. The root logger is used. However, it's best to get into the habit of using specific loggers with different configurations.

Root Logger

The logging module has a hierachy, of which the Root Logger sits at the top.

When we are using the basicConfig() method, we are effectively initializing the root loger. By default, the root logger has a default level of WARNING.

import logging

logging.basicConfig()

logging.debug('Hello this is debug')
logging.info('Hello this is info')
logging.warning('Hello this is warning')

WARNING:root:Hello this is warning

This is also the case when we call a Logger, without specifying the name. Here's how:

import logging

logger = logging.getLogger()

print(f'My name is {logger.name} and my level is {logger.level}.')

My name is root and my level is 30.

Once the root logger is initialized, we cannot add any other logging method to our module.

Why we should not use the root logger

It is not a good idea to directly log against the Root logger. Instead we should create and configure logging entities that serve the logging requriements for each specific module. This is done by logging.getLogger(name).

Let's say we have two Python modules in the same directory:

employee.py:

import logging

logging.basicConfig(
    filename='employee.log',
    level=logging.INFO,
    format='%(levelname)s:%(name)s:%(message)s'
)

class Employee:

    def __init__(self, first, last):
        self.first = first
        self.last = last

        logging.info('Created Employee {} - {}'.format(self.fullname, self.email))

    @property
    def email(self):
        return '{}.{}@email.com'.format(self.first, self.last)

    @property
    def fullname(self):
        return '{} {}'.format(self.first, self.last)


emp1 = Employee('John', 'Smith')

and then we have logbasic.py, which has an import statement:

import employee

logging.basicConfig(filename='sample.log',level=logging.DEBUG,                                          format='%(asctime)s:%(name)s: %(message)s')

        return x + y

        return x - y

        return x * y

        return x/y

num_2 = 5

logging.debug('Mul: {} x {} = {}'.format(num_1, num_2, mul_result))

logging.debug('Div: {} / {} = {}'.format(num_1, num_2, div_result))

When we run logbasic.py, Python will run the script in employee.py.

After logbasic.py is executed, we would realise that there is only one log file: employee.log. This is the log file that was configured in employee.py:

import logging

logging.basicConfig(
    filename='employee.log',
    level=logging.INFO,
    format='%(levelname)s:%(name)s:%(message)s'
)

This root logger was configured here. When it gets to the following line in logbasic.py, the root logger remains unchanged.

logging.basicConfig(filename='sample.log',level=logging.DEBUG,                                          format='%(asctime)s:%(name)s: %(message)s')

So why does the logging from logbasic.py not get printed? Because the logging statements in this file was set at the DEBUG level, where as the root logger has level=logging.INFO.

From this, we can see how sharing a root logger between different modules is bad and confusing.

Logging level and Handlers

Like loggers, we can also set the logging level at the Handler level. This is useful if we want to set multiple handlers for the same logger, but different severity levels for each of them.

If there are two different levels set at the logger as well as a handler that is added to that logger, the one at the handler will take precedence because it handles the actualy writing of the log.

Example:

import logging

logger = logging.getLogger(__name__)
logger.setLevel(logging.INFO)

formatter = logging.Formatter('%(asctime)s:%(name)s: %(message)s')

file_handler = logging.FileHandler('test.log')
file_handler.setFormatter(formatter)
file_handler.setLevel(logging.ERROR)

logger.addHandler(file_handler)


logger.info("Hello World")

The test.log file would be empty because the FileHandler set log level at logging.ERROR where as the logger message was at the logging.INFO level.

The benefit of FileHandler

We can also configure multiple handlers with a logger.

Let's try a new handler: StreamHandler.

  • We'll keep our code the same, just with a new handler that is added to the logger.
  • We are not going to add the file name.
  • We are not going to set a logging level, the new handler will just take the logger level.
import logging

logger = logging.getLogger(__name__)
logger.setLevel(logging.INFO)

formatter = logging.Formatter('%(asctime)s:%(name)s: %(message)s')

file_handler = logging.FileHandler('test.log')
file_handler.setFormatter(formatter)
file_handler.setLevel(logging.ERROR)

stream_handler = logging.StreamHandler()

logger.addHandler(file_handler)
logger.addHandler(stream_handler)

logger.info("Hello World - This is Info")
logger.critical("Oh God!")

try:
    test = 10 / 0 
except:
    logger.exception("This is an error.")
