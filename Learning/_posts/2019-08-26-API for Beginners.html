Quick Introduction

API means Application Programming Interface, i.e an interface used by Programming.

What is an interface? Let's look at a telephone. The interface is the numberpad. This number pad is the "user" interface. When we are talking about API however, we are talking about a machine interface that is used by the computer/ application.

Developers interact with APIs through the user of programming codes. APIs are everywhere on computers.

APIs exposes something useful.


An electric socket is basically an API, which is consumed by appliances. It follows a universal standard to which is brand and model agnostic.

Similarly, APIs are interfaces that are consumed by applications/ programs. APIs are programming language agnostic.

https://programmableweb.com/ has thousands and thousands of APIs for developers.

Example - Build an API

I have a Content Manager called TimeOut. I would like an API that returns GPS coordinates for event venues. TimeOut doesn't know coordinates. It, however knows the venue address.

There is an API for that, which we can utilise.
> Google Geocoding API
(https://developers.google.com/maps/documentation/geocoding/)

10 Downing st, London
> > > 

{
    "location": {
        "lat" : 51.50340666,
        "lng" : -0.1275923
    }
}

This is called a JSON.

How do we do this?

We perform an API call. An API call is an HTTP Request.

https://maps.googleapis.com/maps/api/geocode/json?address=10%20Downing%20st,%20London

(This is no longer free in 2019.)

With the API call, we have the intel we need. We don't need to store this into out database as well. We can if we want.

When do we call an API?

We build our own content manager with a form with venue name + address. The address is then converted into lat and lng via Google Geocoding API (or any other Geocoding API).

The venue name, address and geocoding information will be stored in the database.

Looking at some code

"I want the app to send text message notification when an event is cancelled."

Step 1 - Sign up to Twilio
Step 2 - Fetch Twilio credentials
Step 3 - Write some ruby

require 'twilio-ruby'

account_sid = 'AC______'
auth_token = '________'

client = Twilio::REST::Client.new(account_sid, auth_token)

client.messages(create(
        from: '+0213219421',
        to: '+2142141241',
        body: 'Hi buddy, the event is cancelled!'
        )

Just with 8 lines of ruby, we can send a message to anywhere in the world, using Twilio API. This code is run server-side.

It is very easy to make API work. It is, however, trickier and more timeconsuming to cover all "cornercases".

Webhooks

What's a webhook?

Webhooks work somewhat similarly to hitting the F5 key - Reload the Page.
Let's say, we want to build a feature - Every 10 seconds, call the API to fetch list of Meetup attendees. The retrived JSON is then compared against the previous one for changes.

We want to apply the Hollywood Principle: "Don't call us, we'll call you."
